# OD Press Pilot 操作マニュアル

OD Press Pilot は、WordPress の管理画面から AI を使って「お知らせ本文」「X 用テキスト」「翻訳版 X 用テキスト」「メタディスクリプション」「ハッシュタグ」を作成するためのプラグインです。

事業者ごとの広報プロフィールを登録しておくことで、毎回同じトーンや方針に沿った文章を生成しやすくなります。生成した文章はその場で編集でき、WordPress の投稿下書きとして作成することもできます。

## できること

- 事業者名、サービス概要、強み、NG 表現、SNS 方針などの広報プロフィールを保存できます。
- 投稿したい内容を入力すると、AI がタイトル、本文、X 用テキストなどをまとめて生成します。
- 英語、中国語、韓国語、日本語、任意のカスタム言語の翻訳版 X 用テキストを生成できます。
- よく使う生成条件をテンプレートとして保存できます。
- 生成結果を編集、コピー、再生成できます。
- 生成結果から WordPress の投稿下書きを作成できます。

## 利用前の準備

### 1. プラグインを有効化する

WordPress 管理画面の「プラグイン」から `OD Press Pilot` を有効化します。

有効化すると、管理画面の左メニューに「AIお知らせ作成」が表示されます。

### 2. AI Provider を設定する

OD Press Pilot は、プラグイン内に API キーを保存しません。WordPress の AI Client / Connectors 側で設定済みの AI Provider を利用します。

OpenAI、Google (Gemini)、Anthropic など複数の AI Provider が接続済みの場合は、コンテンツ生成画面の「AI Provider」から今回使用する Provider を選択できます。特に指定しない場合は「AI Client 自動選択」のままで問題ありません。

管理画面で AI Provider が未設定の場合、「AI Provider が利用できません。WordPress 7.0 以上で Settings > Connectors から AI Provider を設定してください。」という警告が表示されます。

その場合は、WordPress 管理画面の `Settings > Connectors` で利用したい AI Provider を設定してください。

### Google (Gemini) 無料枠を利用する場合

Google (Gemini) の無料枠には、RPM（1分あたりのリクエスト数）、TPM（1分あたりの入力トークン数）、RPD（1日あたりのリクエスト数）などの上限があります。

これらの上限は API キー単位ではなく Google Cloud project 単位で適用されます。同じ project を他のサイトやアプリでも利用している場合、OD Press Pilot に表示される回数と Google 側の実際の利用状況は一致しないことがあります。

RPD は Pacific time の午前0時にリセットされます。日本時間の午前0時ではない点に注意してください。

無料枠では、リクエスト内容が Google のサービス改善に使われる場合があります。機密情報、個人情報、公開前の重要情報などは送信しないことをおすすめします。

## 基本的な使い方

最初は、次の順番で使うとわかりやすいです。

1. 「AIお知らせ作成 > 広報プロフィール」で事業者情報を保存する。
2. 必要に応じて「AIお知らせ作成 > テンプレート」でよく使う生成条件を保存する。
3. 「AIお知らせ作成 > コンテンツ生成」で投稿内容を入力して生成する。
4. 生成結果を確認、編集、コピーする。
5. WordPress 投稿として使う場合は「投稿下書き作成」を押す。

## 広報プロフィール

場所: `AIお知らせ作成 > 広報プロフィール`

広報プロフィールは、AI が文章を作るときの前提情報です。ここを丁寧に入力しておくほど、事業者らしい文章を生成しやすくなります。

### 基本プロフィール

入力できる項目は次の通りです。

- 事業者名
- サービス名
- サービス概要
- ターゲット顧客
- 強み・特徴
- 会社理念・想い
- よく使うキャッチコピー
- NG 表現
- よく使う CTA
- SNS 運用方針
- ハッシュタグ方針
- 文章トーン

文章トーンは「丁寧」「カジュアル」「親しみやすい」「フォーマル」から選択できます。

入力後は「保存」を押してください。

### 追加指示

「追加指示」には、毎回 AI に守ってほしいルールを書きます。

例:

```text
本文は読みやすい短めの段落にしてください。
専門用語を使う場合は、一般の方にも伝わる言い換えを添えてください。
価格や日程など、入力情報にない事実は追加しないでください。
```

### プロンプトテンプレート

「プロンプトテンプレート」タブには、広報プロフィールを作るための補助プロンプトがあります。

事業情報がまだ整理できていない場合は、このテンプレートをコピーして AI に貼り付け、必要事項を入力してください。出力された内容を広報プロフィールの各項目へ転記すると、初期設定が進めやすくなります。

## コンテンツ生成

場所: `AIお知らせ作成 > コンテンツ生成`

ここで、お知らせとして発信したい内容を入力して AI 生成を行います。

### 生成条件

主な入力項目は次の通りです。

- 投稿内容: 生成したいお知らせの元情報です。必須項目です。
- 対象読者: 誰に向けたお知らせかを入力します。
- 希望文字数: 本文のおおよその長さを指定します。未入力でも生成できます。
- 翻訳言語: 翻訳版 X 用テキストを作りたい言語を選びます。
- カスタム翻訳言語: 「カスタム入力」を選んだ場合に入力します。
- 絵文字を利用する: X 用テキストなどで絵文字を使いたい場合に有効化します。
- ハッシュタグを生成する: ハッシュタグが必要な場合に有効化します。
- AI Provider: 使用する AI Provider を選びます。OpenAI、Google (Gemini)、Anthropic など複数の AI Provider が接続済みの場合は個別に選択できます。通常は「AI Client 自動選択」で問題ありません。

入力後、「生成」を押すと右側または下部に生成結果が表示されます。

Google (Gemini) を選択している場合は、生成条件の下に「Gemini 利用状況の目安」が表示されます。

ここで表示される「今日の生成回数」「直近1分の生成回数」「最後の上限エラー」は、OD Press Pilot がこの WordPress サイト内で記録した値です。Google AI Studio 側の正確な残り回数や残りトークン数ではありません。正確な利用状況や上限値は Google AI Studio で確認してください。

### 生成結果

生成される項目は次の通りです。

- タイトル
- 本文
- X 用テキスト
- 翻訳版 X 用テキスト
- メタディスクリプション
- ハッシュタグ

各項目は画面上で編集できます。AI の出力をそのまま使わず、公開前に必ず内容を確認してください。

各項目の「コピー」ボタンを押すと、その内容だけをクリップボードへコピーできます。

### 再生成

同じ生成条件でもう一度作り直したい場合は「再生成」を押します。

再生成すると、現在表示されている生成結果が新しい内容に置き換わります。必要な文章がある場合は、先にコピーしておくと安心です。

### 投稿下書き作成

生成結果を WordPress の投稿下書きにする場合は「投稿下書き作成」を押します。

下書き作成に成功すると、編集画面へのリンクが表示されます。投稿編集画面で本文、カテゴリー、アイキャッチ画像、公開日時などを確認してから公開してください。

下書き作成には、生成結果の「タイトル」と「本文」が必要です。

## テンプレート

場所: `AIお知らせ作成 > テンプレート`

テンプレートは、よく使う生成条件を保存しておく機能です。

たとえば、次のような用途に向いています。

- 新商品のお知らせ
- キャンペーン告知
- イベント開催案内
- 営業日変更のお知らせ
- 採用や募集のお知らせ

### 新規作成

1. 「テンプレート」画面を開きます。
2. 「新規作成」を押します。
3. テンプレート名を入力します。
4. 投稿内容、対象読者、希望文字数、翻訳言語、絵文字、ハッシュタグ、AI Provider を設定します。
5. 「保存」を押します。

### 編集

保存済みテンプレートの一覧から編集したいテンプレートを選びます。

内容を変更して「保存」を押すと、テンプレートが更新されます。

### 削除

保存済みテンプレートを選び、「削除」を押します。

確認ダイアログで承認するとテンプレートが削除されます。削除したテンプレートは元に戻せません。

### コンテンツ生成画面で使う

「コンテンツ生成」画面の「保存済みテンプレート」からテンプレートを選び、「適用」を押します。

テンプレートの内容が生成条件に反映されます。必要に応じて投稿内容などを調整してから「生成」を押してください。

「コンテンツ生成」画面で入力した条件を、その場で「テンプレートに保存する」こともできます。

## 生成時のコツ

### 投稿内容には事実を具体的に書く

AI は入力されていない事実を正確には知りません。日付、場所、料金、対象者、申し込み方法など、重要な情報は投稿内容に明記してください。

例:

```text
6月20日から7月15日まで、夏の早期予約キャンペーンを実施します。
対象は公式サイトから宿泊予約をした方です。
特典として、チェックイン時に地域のお菓子セットをプレゼントします。
予約ページのURLは https://example.com/reserve です。
```

### 対象読者を指定する

「初めて利用するお客様」「既存のお客様」「地域の方」「採用応募を検討している方」など、対象読者を指定すると文章のトーンが整いやすくなります。

### NG 表現を登録する

使いたくない言葉、避けたい表現、誇大に見える言い回しがある場合は、広報プロフィールの「NG 表現」に登録しておきます。

### 公開前に必ず確認する

AI が作った文章には、事実と異なる内容や不自然な表現が含まれる場合があります。公開前に、特に次の点を確認してください。

- 日付、時間、価格、場所が正しいか
- URL や問い合わせ先が正しいか
- 誇大表現になっていないか
- 事業者の方針やトーンに合っているか
- X 用テキストが長すぎないか
- ハッシュタグが適切か

## よくあるトラブル

### AI Provider が利用できない

管理画面に AI Provider の警告が表示される場合は、WordPress 側で AI Provider が設定されていません。

`Settings > Connectors` で Provider を設定してから、もう一度「コンテンツ生成」を開いてください。

### コンテンツ生成に失敗する

主な原因は次の通りです。

- AI Provider の API キーが未設定、または無効になっている。
- AI Provider の利用上限や課金設定に問題がある。
- AI Provider 側で一時的なエラーや混雑が発生している。
- 投稿内容が未入力になっている。
- AI のレスポンスが期待した JSON 形式にならなかった。

まず、投稿内容が入力されているか、AI Provider の設定が有効かを確認してください。

`Service Unavailable (503)` や `currently experiencing high demand` が表示される場合は、AI Provider 側の混雑または一時障害です。数分から十数分ほど時間を空けて再実行してください。別のモデルを選べる場合は、モデルを切り替えることで改善することがあります。

### Google (Gemini) の rate limit / quota に達した可能性がある

`429`、`RESOURCE_EXHAUSTED`、`quota exceeded`、`rate limit` などが表示される場合は、Google (Gemini) の無料枠または現在の利用上限に達した可能性があります。

短時間の上限に達している場合は、少し時間をおいてから再実行してください。日次上限に達している場合は、Pacific time の午前0時以降に再度お試しください。継続的に利用する場合は、Google AI Studio で rate limit、quota、課金設定を確認してください。

コンテンツ生成画面で Google (Gemini) を選択している場合、OD Press Pilot はこのサイトからの生成回数と最後に検知した rate limit / quota 系エラーを表示します。ただし、この表示は OD Press Pilot 内の記録であり、Google 側の正確な残量ではありません。

### 下書き作成に失敗する

生成結果の「タイトル」または「本文」が空の場合、下書きは作成できません。

タイトルと本文を入力または再生成してから、もう一度「投稿下書き作成」を押してください。

### 翻訳版 X 用テキストが出ない

翻訳言語を選んでいない場合、翻訳版 X 用テキストは生成されません。

翻訳が必要な場合は、「翻訳言語」で必要な言語を選択してから生成してください。

## 外部プラグインからテンプレートを追加する

OD Press Pilot が公開する公式フックは、短い `odpp_` prefix を使います。

別プラグインから `odpp_external_templates` フィルターを使うと、OD Press Pilot 本体の保存済みテンプレートとは別にテンプレートを追加できます。

```php
add_filter(
	'odpp_external_templates',
	static function ( array $templates ): array {
		$templates[] = [
			'id'                    => 'example_campaign_summer_2026',
			'name'                  => '夏キャンペーン告知',
			'post_content'          => 'キャンペーン内容、期間、特典、申込方法を入力してください。',
			'audience'              => '既存顧客と新規顧客',
			'desired_length'        => 800,
			'translation_languages' => [ 'en' ],
			'use_emoji'             => true,
			'generate_hashtags'     => true,
			'provider'              => 'auto',
		];

		return $templates;
	}
);
```

外部テンプレートは `GET /od-press-pilot/v1/templates` に含まれ、コンテンツ生成画面で選択して生成条件へ適用できます。一方で、`od_press_pilot_templates` option には保存されず、テンプレート管理画面では編集・削除できない読み取り専用テンプレートとして表示されます。

`id` が保存済みテンプレートや他の外部テンプレートと重複した場合、その外部テンプレートは読み込まれません。テンプレートパックを作る場合は、プラグイン固有の prefix を付けた ID を使ってください。

## 権限について

- 「コンテンツ生成」と「テンプレート」は、投稿を編集できるユーザーが利用できます。
- 「広報プロフィール」は、管理者権限を持つユーザーが利用できます。

## 管理者向けメモ

- 対応 WordPress: 7.0 以上
- 対応 PHP: 7.4 以上
- プラグインバージョン: 0.1.5
- REST API namespace: `od-press-pilot/v1`
- 広報プロフィール保存先 option: `od_ai_writer_profile`
- テンプレート保存先 option: `od_press_pilot_templates`
- API キーはこのプラグインでは保存しません。
- GitHub Releases 経由の更新確認には `inc2734/wp-github-plugin-updater` を利用します。

ローカル開発環境では、次の URL と認証情報を使います。

- WordPress: `http://localhost:8890`
- 管理画面: `http://localhost:8890/wp-admin`
- ユーザー名: `admin`
- パスワード: `password`

よく使う開発コマンド:

```bash
npm install
composer install
npm run build
npm run env:start
npm run wp -- plugin list
npm run lint:php
```
